## 2026-03-07 - Codebase-Specific DB Query Bottlenecks
**Learning:** The application codebase has a recurring anti-pattern where data insertion routes (e.g., `/insert` in `bbsController.js` and `refController.js`) perform redundant, unutilized O(N) database queries like `.count()` and `.findAll()` immediately before responding with a redirect or render. Additionally, `/getContent` routes were using `.findAndCountAll()` when only the data (and not the total count) was being sent to the client.
**Action:** When working on Express route controllers, carefully inspect data flow to identify and remove `findAll()` and `count()` database calls if their results are discarded or unutilized before ending the HTTP response. Prefer `findAll()` over `findAndCountAll()` when pagination total counts are not needed.
