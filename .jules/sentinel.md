## 2025-03-02 - [CRITICAL] Fixed Path Traversal / LFI in Move Routers
**Vulnerability:** The dynamic view routers `routes/move.js` and `routes/enMove.js` passed completely unsanitized, user-provided `req.params.dir` and `req.params.file` directly into `res.render(req.params.dir + "/" + req.params.file)`. This allowed attackers to use directory traversal sequences (e.g. `../../`) and URL encoded paths (`%2e%2e%2f`) to access arbitrary `.js`, `.json`, or templates on the server, potentially leading to LFI or RCE depending on EJS features and configuration.
**Learning:** This architectural pattern (using catch-all routes like `/:dir/:file` to render dynamic pages without validation) is inherently dangerous in Express applications, particularly when using `res.render`.
**Prevention:** Implement a strict whitelist of allowed view directories (e.g., `['about', 'contact', ...]`) and strictly enforce filename structure using regex (e.g., `/^[a-zA-Z0-9_\-\.]+$/`) in a shared `validatePath` middleware to block traversal characters.
