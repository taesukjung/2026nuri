## 2024-05-28 - Replace unsemantic save/del anchors with accessible buttons
**Learning:** Legacy form interactions relying on `javascript:func()` inside anchor tags lack native button semantics, keyboard focus visibility natively, and are hard to augment with complex states (like `disabled`). Attempting to add "disabled" visual states to anchors requires extra CSS tricks and doesn't actually prevent keyboard activation.
**Action:** Always migrate `<a href="javascript:submit()">` to `<button type="button" onclick="submit()">`. Apply native `disabled` attributes alongside text changes (e.g., "Saving...") to provide robust screen reader feedback and prevent double-submission natively without hacky CSS `pointer-events: none` implementations. Ensure CSS class selectors (e.g., `.btn_area a`) are updated to handle `.btn_area button` uniformly to prevent visual regression.
