## 2024-05-14 - Redundant Database Queries Before View Renders
**Learning:** In this codebase, operations like `insert` in the `bbsController.js` and `refController.js` were executing redundant `.count()` and `.findAll()` Sequelize queries immediately before calling `res.redirect()` or `res.render()`. Because the fetched data was completely unused (e.g., only the count was logged to console), this created a significant O(N) database bottleneck on every write operation, slowing down request completion times unnecessarily.
**Action:** Always verify that data fetched from the database immediately prior to a redirect or render is actually passed to the view or used in subsequent logic. If the data is unused, the query should be removed to eliminate unnecessary database load and improve response times.
